# Security Policy for UniNest

## Supported Versions

The following table describes which versions of this project currently receive security updates:

| Version | Supported |
|---------|-----------|
| Latest stable release | Yes |
| Previous minor release | Limited |
| Older releases | No |

Only the latest stable release is guaranteed to receive timely security patches. Users are strongly encouraged to upgrade to the most recent version.

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly using one of the following methods:

- Email: arqam.mousa@gmail.com
- Private message to the project maintainer (if applicable)

Please do not open public GitHub issues for security vulnerabilities.

When reporting, include:
- A detailed description of the vulnerability
- Steps to reproduce
- Affected versions
- Any proof-of-concept code or screenshots (if available)

## Response Process

After a report is received:

- You will receive an acknowledgment within 72 hours
- The issue will be reviewed and validated
- If accepted, a fix will be developed and released
- You may be contacted for additional information
- Public disclosure will occur only after a fix is available

## Security Updates

Security patches will be released as part of regular releases or as hotfixes for critical vulnerabilities.

## Scope

This policy applies to:
- Backend services
- APIs
- Web and mobile applications
- Infrastructure and deployment configurations (if applicable)

## Safe Harbor

We support responsible disclosure and will not take legal action against researchers who:
- Make a good-faith effort to avoid privacy violations and data destruction
- Do not exploit the vulnerability beyond proof of concept
- Do not publicly disclose the issue before a fix is released
